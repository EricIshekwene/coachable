d-breakpoints

01 — Breakpoints: Four thresholds, mobile-first

Write base styles for mobile. Layer md:, lg:, xl: overrides on top. Never write desktop-first and undo at small sizes.

Mobile-first means the base layer is the most constrained. Expanding outward is additive. Collapsing inward requires overriding every property — a maintenance cost that compounds.

d-layout

02 — Layout: Sidebar + content column, contained at 2xl

Sidebar 240px · Content column max 800px for reading · Right panel 360–480px at xl · Wrap in max-w-[1440px] at 2xl.

Reading lines over 80 characters cause fatigue. The max-width container is not a constraint on ambition — it is a legibility decision.

Layout at lg — three zones
Sidebar 240px Content column max 800px Right panel 360px @ lg · 480px @ xl 2xl container max-w-[1440px] mx-auto
d-margins

03 — Margins: Scale with the viewport

16px mobile · 24px md · 32px lg · 40px xl. Section gaps stay 24px at all breakpoints — they do not stretch on wide screens.

Margins scale with screen real estate. Section gaps don't — widening the space between sections on a large monitor creates visual disconnection, not breathing room.

d-hover

04 — Hover States: Required on all interactive elements

Every clickable element needs a :hover style. Hover = subtle background tint. Never color change (except sidebar nav). Never the sole discovery mechanism for a feature.

Without hover feedback, interactive elements look inert. A list row with no hover state looks like a static label — the user hesitates, and trust in the interface erodes.

List row — three states
Default — no background default
Hovered — BrandBlack2 tint :hover
Selected — BrandOrange (general standard) active
d-cursor

05 — Cursor: Pointer on every clickable element

cursor: pointer on all clickable elements. grab / grabbing on draggable. not-allowed on disabled. default on text and non-interactive containers.

An element that looks interactive but shows a text cursor fails the affordance test. The cursor communicates intent before the click.

Cursor types by context
👆 pointer Buttons, links, rows, nav items
✋ grab / grabbing Draggable elements
↔ ew-resize Resizable panel edges
🚫 not-allowed Disabled elements
I default / text Non-interactive, text content
✕ never Default cursor on a clickable div
d-nav

06 — Navigation: Left sidebar on desktop, never bottom bar

Sidebar 240px expanded · 64px icon-only collapsed · Three nav states: inactive (BrandGray), hover (BrandText + tint), active (BrandOrange, no pill).

Bottom nav is within thumb reach on mobile. On desktop, the thumb is not the input device. The sidebar uses the available vertical space and keeps the content area unobstructed.

d-dialog

07 — Dialogs: Centered and dismissible, not bottom sheets

On md+, bottom sheets become centered dialogs. Always: X close button + Escape dismiss + focus trap + focus restore on close.

Dialogs without Escape support force mouse usage on keyboard-only users. Focus traps without restoration disorient screen reader users.

d-drawer

08 — Panels & Drawers: Right panel for secondary context

360px at lg · 480px at xl · Slides in from right (transform + opacity, 200ms) · Dismiss via X or Escape · 1px left border separation.

Panels keep the user oriented in the primary view while showing detail. Navigating to a new page for every selection breaks spatial memory.

d-density

09 — Data Density: Desktop views can be tighter

Table row padding: 8px (vs 12px mobile) · List row height: 40px (vs 48px) · Card gap: 12px (vs 16px) · Section gaps: 24px always.

Desktop users see more content at once. Reducing spacing in data-heavy views is correct and expected — it matches the precision of a pointer device.

d-tooltip

10 — Tooltips: Valid on desktop, never the sole source of information

BrandBlack2 background · BrandText caption text · radius-sm 6px · max-width 200px · appears after 400ms hover · never the only way to know what a control does.

Tooltips are invisible to touch users and keyboard-only users who tab but don't trigger hover. If a feature is only explained in a tooltip, it's hidden from half your users.
